Architecture cible
Frontend

PostgreSQL + pgvector — métadonnées & embeddings
IA & génération
OpenAI GPT-4o, DALL·E 3, Embeddings API
Répartition des tâches
T1
Pipeline de données — Cleaning & Embeddings
Responsable : Membre 1
Collecte & nettoyage : récupération du dataset Kaggle Movies, normalisation des champs (titre, genre, synopsis, note, année), suppression des doublons.
Génération d'embeddings : vectorisation de chaque synopsis via OpenAI text-embedding-3-small, insertion dans PostgreSQL + pgvector avec index HNSW.
Pipeline reproductible : script Python automatisé pour mise à jour et rechargement du dataset sans intervention manuelle.
Python / Pandas OpenAI Embeddings PostgreSQL + pgvector Kaggle Dataset
T2
Génération d'images — Affiches de films
Responsable : Membre 2
Prompt builder : GPT-4o extrait automatiquement genre, ambiance et époque de chaque film pour construire un prompt artistique générique, contournant les restrictions copyright de DALL·E.
Génération & stockage : création des affiches via DALL·E 3 / gpt-image-1, mise en cache sur Cloudflare R2 pour éviter toute régénération inutile.
Fallback visuel : en cas de refus de l'API, génération automatique d'une affiche stylisée (titre + couleurs de genre) via Canvas API.
DALL·E 3 GPT-4o Cloudflare R2 Canvas API
T3
Moteur de recherche & recommandation
Responsable : Membre 3
Recherche hybride : combinaison full-text search (PostgreSQL tsvector) et recherche vectorielle (cosine similarity pgvector) pour des résultats précis et contextuels.
Extraction de critères : GPT-4o analyse le prompt libre de l'utilisateur et extrait genre, année, humeur → filtres SQL dynamiques.
Ranking & orchestration : score combiné (similarité vectorielle + note + fraîcheur), orchestré via LangChain pour une recommandation pertinente et ordonnée.
FastAPI LangChain pgvector similarity GPT-4o
T4
Frontend complet & authentification utilisateurs
Responsable : Membre 4
Design & pages : interface complète — /movies (catalogue), /recommenderFilms (chat de recommandation), /movie/:id (détail + affiche générée). Design system cohérent Tailwind CSS.
Authentification : système complet login/register avec NextAuth.js — email/password + OAuth Google — sécurisé par JWT côté FastAPI.
Performance & UX : streaming des résultats en temps réel (SSE), skeleton loaders, lazy loading des affiches, CDN pour les assets, historique des recherches utilisateur.
Next.js 14 Tailwind CSS NextAuth.js SSE streaming
T5
Intégration globale de l'application — Jaril
Responsable : Jaril
Assemblage des modules : Jaril récupère chaque module livré (pipeline, images, moteur, frontend) et les intègre dans une application unifiée, cohérente et pleinement fonctionnelle.
API Gateway FastAPI : définition et validation des routes /prompteUser, /movies, /recommenderFilms — contrats d'API TypeScript auto-générés (OpenAPI → ts-rest) pour garantir la cohérence front/back.
Orchestration LangGraph : coordination de l'agent images et de l'agent recommandation en parallèle, avec gestion des erreurs et fallbacks à chaque étape.
Tests & déploiement : tests end-to-end Playwright sur les parcours critiques, logs structurés, health checks, déploiement final de l'application complète.
FastAPI + Pydantic LangGraph OpenAPI / ts-rest Playwright Docker
